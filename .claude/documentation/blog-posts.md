# Blog Posts

## File Location and Naming

Posts live in `_posts/` with the naming convention:

```
YYYY-MM-DD-slug.html
```

Example: `2026-03-28-welcome.html`

The date in the filename determines the post's publication date and URL. The slug becomes the URL-friendly title portion.

## Required Frontmatter

```yaml
---
layout: post
title: "Post Title Here"
date: 2026-03-28
author: "Teresa Ristow"
---
```

**Note:** `layout: post` and `author: "Teresa Ristow"` are set as defaults in `_config.yml` for all files in `_posts/`, so they can be omitted. However, `title` and `date` are always required.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `layout` | string | No | `post` | Layout template |
| `title` | string | Yes | - | Post title (appears in h1, meta, feeds) |
| `date` | date | Yes | - | Publication date (YYYY-MM-DD) |
| `author` | string | No | `Teresa Ristow` | Author name |

## Optional Frontmatter

| Field | Type | Description |
|-------|------|-------------|
| `excerpt` | string | Custom excerpt for meta description and post cards. If omitted, Jekyll auto-generates from first paragraph. |
| `categories` | list | Post categories (affects URL if permalink uses `:categories`) |
| `tags` | list | Post tags for filtering/display |
| `image` | string | Featured image path (used by jekyll-seo-tag for OG image) |

## Content Format

Posts use **HTML**, not Markdown. Write content directly in HTML tags:

```html
---
layout: post
title: "My Post Title"
date: 2026-04-01
excerpt: "A brief description for previews and meta tags."
---

<p>First paragraph of the post.</p>

<h2>Section Heading</h2>
<p>More content here.</p>

<ul>
  <li>List item one</li>
  <li>List item two</li>
</ul>
```

Content area styles (lists, headings, images) are scoped to `.post__content` in the CSS.

## Embedding React Components

To add interactive components to a post:

1. Include the React mount script (once per post, before any component divs):
   ```html
   {% include react-mount.html %}
   ```

2. Add component mount points:
   ```html
   <div data-component="sample-chart" data-props='{"label": "Results", "values": [72, 85, 91]}'></div>
   ```

3. The component name must match a file at `assets/js/components/{name}.js`.

See `js-and-react-components.md` for component authoring details.

## How Posts Appear

| Location | What Shows | Logic |
|----------|-----------|-------|
| `/blog/` page | All posts | `{% for post in site.posts %}` with post-card.html |
| Homepage | 3 most recent | `{% for post in site.posts limit:3 %}` with post-card.html |
| RSS feed | All posts | Auto-generated by jekyll-feed at `/feed.xml` |
| Sitemap | All posts | Auto-generated by jekyll-sitemap at `/sitemap.xml` |

## Permalink Structure

Configured in `_config.yml`:

```yaml
permalink: /blog/:year/:month/:title/
```

Example: A post named `2026-03-28-welcome.html` gets the URL:
```
/blog/2026/03/welcome/
```

## Drafts

Place draft posts in `_drafts/` (create the directory if needed):

```
_drafts/
  my-upcoming-post.html    (no date prefix needed)
```

Drafts are not published to the live site. To preview locally, run `jekyll serve --drafts`. When ready to publish, move the file to `_posts/` and add the date prefix.

## SEO

- **Meta description:** The `excerpt` frontmatter field becomes the meta description via `jekyll-seo-tag`. If no excerpt is set, Jekyll auto-extracts from the first paragraph.
- **Open Graph / Twitter Cards:** Auto-generated by `jekyll-seo-tag`. Set `image` in frontmatter for a custom OG image; otherwise falls back to `og_image` in `_config.yml`.
- **JSON-LD:** `json-ld.html` outputs `BlogPosting` schema for all posts, including headline, datePublished, author, description, and URL.
- **Canonical URL:** Auto-generated by `jekyll-seo-tag`.

## Checklist for New Posts

1. Create file: `_posts/YYYY-MM-DD-slug.html`
2. Add frontmatter: `title`, `date`, `excerpt` (recommended)
3. Write HTML content
4. If using React components: add `{% include react-mount.html %}` before component divs
5. Verify: post appears on `/blog/` page and homepage (if among latest 3)
6. No em-dashes in content (workspace constraint)
